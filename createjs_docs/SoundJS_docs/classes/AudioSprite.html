AudioSprite 类

定义: AudioSprite:6
模块: SoundJS

0.6.0 及以上版本

注意:AudioSprite不是一个类,但其使用文档中很容易丢失,所以它被称为快速参考这里。

音频精灵很像CSS精灵或图像精灵表：组合成一个单一的文件多音频资产。音频精灵变通限制在某些浏览器， 其中只有一个单一的声音可以被加载并在一个时间播放。我们建议的音频片段之间的空挡至少300ms的处理HTML标签音频误差，并防止邻近的剪辑不慎打位。

音频精灵的优点：

  • 旧的浏览器和设备，只允许单个音频实例，如iOS 5的更强大的支持。
  • 它们提供了一个变通的Internet Explorer 9的音频标签的限制，这限制了可一次装入多少不同的声音。
  • 只需要几个声音一个网络请求，尤其是在那里每个文件的网络往返可以添加显著延迟的移动设备更快的加载。

音频精灵的缺点

  • 使用HTML或Flash音频时平稳循环的任何保证。如果你有一个需要循环顺畅的轨道，并要支持非Web浏览器的音频，不使用音频精灵的声音，如果你能避免它。
  • 不能保证HTML声音会立即回放，特别是第一次。在某些浏览器（！镀铬），HTML音频将只加载通过足以在当前的下载速度播放 - 所以我们依靠canplaythrough事件，以确定是否音频被加载。由于音频精灵必须直接跳到扮演着特定的声音，声音可能还没有完全下载。
  • 音频精灵共享相同的核心来源，所以如果你有5声音的精灵，并只限于2同时执行的情况下，你只能在同一时间播放声音的2。

实例

    createjs.Sound.initializeDefaultPlugins();
    var assetsPath = "./assets/";
    var sounds = [{
        src:"MyAudioSprite.ogg", data: {
            audioSprite: [
                {id:"sound1", startTime:0, duration:500},
                {id:"sound2", startTime:1000, duration:400},
                {id:"sound3", startTime:1700, duration: 1000}
            ]}
        }
    ];
    createjs.Sound.alternateExtensions = ["mp3"];
    createjs.Sound.on("fileload", loadSound);
    createjs.Sound.registerSounds(sounds, assetsPath);
    // 之后加载完成
    createjs.Sound.play("sound2");

您还可以通过创建一个新的AbstractSoundInstance时设置startTime和持续时间动态创建音频精灵。 

    createjs.Sound.play("MyAudioSprite", {startTime: 1000, duration: 400});

优秀的CreateJS社会创造了在工具来创建音频精灵，可 https://github.com/tonistiigi/audiosprite, 以及一个 jsfiddle 于输出到SoundJS格式转换。
