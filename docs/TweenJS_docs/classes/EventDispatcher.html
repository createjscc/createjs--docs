EventDispatcher类

定义于: EventDispatcher:41
模块: CreateJS

EventDispatcher提供了一些方法来管理队列的事件监听器和调度事件。

你可以通过EventDispatcher或混合的方法扩展到现有的原型或通过使用实例EventDispatcher的initialize 方法。

和CreateJS事件类一起,EventDispatcher提供了一个扩展的基于事件模型 DOM级别2事件模型,包括addEventListener， removeEventListener,dispatchEvent。 它支持 bubbling (冒泡)/ capture（捕获）、preventDefault stopPropagation stopImmediatePropagation,handleEvent。

EventDispatcher公开了一个 on 方法, 这使得它更容易创建范围监听器,监听器和监听器相关的任意数据只运行一次。这个 off 方法只不过是 removeEventListener 方法的一个别名。

另一个DOM2级模型 removeAllEventListeners 方法,该方法可用于对所有事件监听器, 或为一个特定的事件监听器。 这个事件对象包括一个可消除活跃的侦听器 remove 方法。

例如

"MyClass"类添加EventDispatcher功能。

 EventDispatcher.initialize(MyClass.prototype);

添加一个事件(见 addEventListener). 

 instance.addEventListener("eventName", handlerMethod);
    function handlerMethod(event) {
    console.log(event.target + " Was Clicked");
    }

保持适当的范围
范围 (ie. "this") 将会随着事件变化. 使用这个on 方法去简化了这个订阅事件.

 instance.addEventListener("click", function(event) {
    console.log(instance == this); // 错误的,范围是模棱两可的。
    });

    instance.on("click", function(event) {
    console.log(instance == this); // 真的,"on"使用了默认调度范围
    });

如果你想使用addEventListener代替,您可能要使用function.bind()或类似的代理来管理范围。

浏览器支持 CreateJS的这个事件模型可以使用单独的套用在任何项目中,然而继承模型需要现代浏览器IE9(+)。

添加特定的事件监听器。注意,添加多个听众相同的函数将导致多个回调被解雇。

例如

 displayObject.addEventListener("click", handleClick);
    function handleClick(event) {
    // 单击发生。
    }

参数:

  • type String

    字符串类型的事件.

  • listener Function | Object

    一个拥有handleEvent方法的对象，或者一个当这个事件被分派时将会被回调的函数。

  • [useCapture] Boolean optional

    对于冒泡的事件,表明是否侦听事件捕获目标或冒泡阶段。

返回:

Function | Object:

返回链接的侦听器或任务。

给所有监听器注册特定的事件。

 // 使用一个字符串事件

    当一个字符串传递给eventObj指定了bubbles(冒泡)值。

  • [cancelable] Boolean 可选的

    当一个字符串传递给eventObj指定了cancelable(可取消的)值.

Boolean:

on

(
  • type
  • listener
  • [scope]
  • [once=false]
  • [data]
  • [useCapture=false]
)
Function

定义于 on:187

使用addEventListener方法的一个快捷方式,可以更容易地指定一个执行范围,有一个侦听器只运行一次,将任意数据作为侦听器的参数,和删除这个侦听器。

这种方法是通过创建一个匿名的包装函数和用addEventListener订阅它。 使用的包装函数返回removeEventListener (或者 off)。

重要的是: 使用on删除一个侦听器,你必须通过包装器函数返回侦听器,或使用remove。 同样地,每一次调用on一个新的包装器函数将被订阅,所以用相同的参数多个调用on将创建多个侦听器。

例如

 var listener = myBtn.on("click", handleClick, null, false, {count:3});
    function handleClick(evt, data) {
    data.count -= 1;
    console.log(this == myBtn); // true——默认为调度范围
    if (data.count == 0) {
    alert("clicked 3 times!");
    myBtn.off("click", listener);
    // 替换: evt.remove();
    }
    }

参数:

  • type String

    字符串类型的事件。.

  • listener Function | Object

    一个有handleEvent方法对象,或一个在事件注册时被回调的函数。

  • [scope] Object 可选的

    执行侦听器的范围。 默认为dispatcher/ currentTarget函数监听器,和侦听器本身为对象的监听器(ie:使用handleEvent)。

  • [once=false] Boolean optional

    如果为true,这个监听器在第一次触发后移除自己。

  • [data] 可选的

    任意的数据将作为第二个参数调用侦听器。

  • [useCapture=false] Boolean 可选的

    对于冒泡的事件,表明是否侦听事件捕获目标或冒泡阶段

返回:

Function:

返回的匿名函数作为侦听器被创建并分配。 这使用之后需要用.removeEventListener删除侦听器。

定义于 removeAllEventListeners:277

删除指定类型的所有监听器,或所有监听类型。

例如

 // 移除所有的监听器
    displayObject.removeAllEventListeners();

    // 移除所有的点击监听
    displayObject.removeAllEventListeners("click");

参数:

  • [type] String 可选的

    字符串类型的事件。 如果省略了,所有的监听类型将被删除。
